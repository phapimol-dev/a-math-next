## A-Math Multiplayer

เว็บเกมกระดาน A-Math แบบออนไลน์ เล่นได้ทั้งกับเพื่อนและบอท (AI) บนกระดานขนาด 15×15 ช่อง คล้ายเกมต่อคำไขว้ แต่ใช้ตัวเลขและสัญลักษณ์ทางคณิตศาสตร์เพื่อสร้าง "สมการที่ถูกต้อง" แข่งกันทำคะแนนให้ได้มากที่สุด

### สิ่งที่โปรเจกต์นี้ทำได้

- **เล่นกับเพื่อนแบบออนไลน์ (1v1)**: สร้างห้องและแชร์รหัสให้เพื่อนเข้ามาเล่นด้วยกันผ่าน Socket.IO
- **เล่นคนเดียวกับบอท (AI)**: เลือกระดับความยากได้ 3 ระดับ (L1, L2, L3) พร้อมระบบคิดสมการอัตโนมัติ
- **ห้อง Public / Private**: 
  - **Public**: แสดงในหน้า Lobby ให้คนอื่นกดเข้าร่วมได้ทันที  
  - **Private**: ใช้รหัสห้อง (Room Code) เพื่อเข้าร่วมเท่านั้น
- **ระบบ Challenge / Manual Validation**: เปิดโหมดให้ตัดสินสมการด้วยคน (แทนที่จะใช้ตัวตรวจสอบอัตโนมัติอย่างเดียว)
- **กระดานพิเศษแบบ A-Math แท้**:
  - `TE` – Triple Equation (สมการ 3 เท่า)
  - `DE` – Double Equation (สมการ 2 เท่า)
  - `TP` – Triple Piece (เบี้ย 3 เท่า)
  - `DP` – Double Piece (เบี้ย 2 เท่า)
- **กติกา A-Math เต็มรูปแบบ**:
  - ชุดเบี้ย 100 ตัวตามมาตรฐาน A-Math จริง (`src/lib/tiles.js`)
  - ตรวจว่าสมการถูกต้องด้วยตัวตรวจสมการ (`src/lib/math_validator.js`)
  - ระบบคิดคะแนน + โบนัส Bingo 40 คะแนน เมื่อใช้เบี้ย ≥ 8 ตัวในตาเดียว
- **รองรับ 2 ภาษา (TH / EN)**: สลับภาษาอินเทอร์เฟซได้ในหน้าเกม (`src/lib/translations.ts`)
- **เสียงเอฟเฟกต์และ UI ทันสมัย**: มีเสียงเมื่อผลัดตา ชนะ/แพ้ และดีไซน์แบบ glassmorphism

---

### เทคโนโลยีที่ใช้

- **Next.js 14 (App Router)** – ส่วน UI หน้าเว็บ (`src/app`, `src/components`)
- **React 18** – สร้าง UI และ state ของเกมฝั่ง client
- **Express + Socket.IO** – รัน backend server, ห้องเกม, ระบบเทิร์น, การคิดคะแนน (`server.mjs`)
- **TypeScript** – ส่วนใหญ่ของโค้ดฝั่ง UI
- **Tailwind CSS + CSS เพิ่มเติม** – สไตล์ UI (`globals.css`, `Home.css`, `Game.css`)

---

### โครงสร้างโปรเจกต์ (สั้น ๆ)

- `server.mjs` – HTTP + WebSocket server (Express + Socket.IO) และการเชื่อมกับ Next.js
- `src/app/page.tsx` – หน้าแรกของแอป, สลับระหว่างหน้าล็อบบี้ (`Home`) และหน้าเกม (`Game`)
- `src/components/Home.tsx` – Lobby, สร้างห้อง, เลือก Public/Private, เลือกเล่นกับบอท, แสดง Public Lobbies
- `src/components/Game.tsx` – กระดาน, หมาก, การส่งสมการ, เปลี่ยนเบี้ย, ยอมแพ้, แสดงเวลา/คะแนน/ผู้ชนะ
- `src/lib/tiles.js` – กำหนดชุดเบี้ย A-Math อย่างเป็นทางการ และฟังก์ชันสุ่ม/จั่วเบี้ย
- `src/lib/math_validator.js` – ตรวจว่าสมการบนกระดาน "ถูกต้องตามคณิตศาสตร์"
- `src/lib/bot_ai.js` – อัลกอริทึมสำหรับหาตาที่ดีที่สุดให้บอท (ขึ้นกับระดับความยาก)
- `src/lib/translations.ts` – ข้อความ UI แบบ 2 ภาษา (TH/EN)

### การติดตั้งและรันโปรเจกต์

#### 1. ความต้องการเบื้องต้น

- **Node.js** แนะนำ **>= 18**  
- **npm** (มาพร้อม Node.js อยู่แล้ว)

ตรวจเวอร์ชัน:

```bash
node -v
npm -v
```

#### 2. ติดตั้ง dependencies

จากโฟลเดอร์โปรเจกต์ (ที่มี `package.json`):

```bash
npm install
```

#### 3. รันโหมดพัฒนา (Development)

คำสั่ง:

```bash
npm run dev
```

คำสั่งนี้จะ:

- รัน `server.mjs` ซึ่งจะ:
  - สตาร์ท Express + Socket.IO backend
  - โหลด Next.js (โหมด dev) แล้วผูก route ให้
- ค่าเริ่มต้นจะฟังที่พอร์ต `3000`

เปิดเบราว์เซอร์ไปที่:

- `http://localhost:3000`

#### 4. รันโหมด Production (Build + Start)

1. **Build ตัว Next.js ก่อน**

```bash
npm run build
```

2. **Start server ในโหมด production**

```bash
npm run start
```

คำสั่ง `start` จะเซ็ต `NODE_ENV=production` และรัน `server.mjs` ซึ่งจะ:

- ใช้ไฟล์ build ของ Next.js จากโฟลเดอร์ `.next`
- รัน Express + Socket.IO สำหรับเกมเหมือนเดิม

คุณสามารถกำหนดพอร์ตเองได้โดยตั้ง environment variable:

```bash
PORT=4000 npm run start
```

แล้วเปิด `http://localhost:4000`

#### 5. โหมด Pure Backend (ไม่รัน Next.js)

ไฟล์ `server.mjs` มีโหมด **Pure Backend** สำหรับรันเฉพาะ Socket.IO + REST เล็กน้อย (เช่น `/health`) โดยไม่โหลด Next.js  

เซ็ต environment:

```bash
PURE_BACKEND=true npm run dev
```

หรือหากไม่มีโฟลเดอร์ `.next` อยู่ (ยังไม่ build) จะเข้าสู่โหมดนี้อัตโนมัติ

### ภาพรวมการใช้งานเว็บ

#### 1. หน้า Lobby (`Home`)

- **ตั้งชื่อผู้เล่น (Player Name)** – ชื่อที่จะแสดงในห้องและหน้าเกม
- เลือกประเภทห้อง:
  - **Public Room** – ห้องสาธารณะ เห็นได้จากทุกคนในรายการ “Public Lobbies”
  - **Private Room** – ห้องส่วนตัว ต้องใช้รหัสห้อง (Room Code) เพื่อเข้าร่วม
- **Create Human Room** – กดสร้างห้องเล่นกับมนุษย์:
  - เลือก Public / Private ตามปุ่มด้านบน
- **Play vs AI (Singleplayer)** – เลือกระดับความยากของบอท:
  - Level 1 – ง่าย
  - Level 2 – ปานกลาง
  - Level 3 – ยาก (เน้นหาตาที่ดีที่สุด)
- **Challenge Mode (Manual Validation)**:
  - หากติ๊กเปิดไว้ ห้องจะใช้โหมดตรวจสมการแบบ manual (อิงจากกติกาเกมที่ต้องตัดสินกันเอง)
- **เข้าห้องด้วยรหัส (Room Code)**:
  - ใส่รหัสห้องเช่น `X1Y2Z3` แล้วกด **Join**
- **Public Lobbies**:
  - รายการห้อง Public ที่ยังรอผู้เล่นอยู่
  - กดปุ่ม **Join Room** เพื่อเข้าทันที

#### 2. หน้าเกม (`Game`)

- **กระดาน 15×15**:
  - วางเบี้ยตัวเลขและสัญลักษณ์ทางคณิตศาสตร์เพื่อสร้างสมการตามกติกา
  - ช่องสีพิเศษ:
    - `TE` – สมการทั้งชุดได้คะแนน ×3
    - `DE` – สมการทั้งชุดได้คะแนน ×2
    - `TP` – ตัวเบี้ยช่องนั้นได้คะแนน ×3
    - `DP` – ตัวเบี้ยช่องนั้นได้คะแนน ×2
- **ถุงเบี้ย (Bag)**:
  - แสดงจำนวนเบี้ยที่เหลือในถุง
- **หมากในมือ (Your Rack)**:
  - คลิกเลือกเบี้ย แล้วคลิกช่องบนกระดานเพื่อวาง
  - หากเป็นเบี้ยเปล่า (Blank) จะมีหน้าต่างให้เลือกค่าที่ต้องการแทน เช่น `0-9`, `+`, `-`, `*`, `/`, `=`
- **การส่งตาเดิน (Submit Move)**:
  - ปุ่ม **Submit Move** จะทำการส่งสมการไปยัง server
  - เซิร์ฟเวอร์จะ:
    - ดึงสมการจากกระดาน (`extractEquations`)
    - ตรวจว่าสมการทุกตัวถูกต้อง (`validateEquation`)
    - คำนวณคะแนน (`calculateScore`) + โบนัส Bingo ถ้าใช้เบี้ย ≥ 8 ตัว
- **ล้างตา (Clear)**:
  - คืนเบี้ยที่เพิ่งวางกลับมาที่ Rack
- **เปลี่ยนเบี้ย (Swap Tiles)**:
  - เปิดโหมด Swap แล้วเลือกเบี้ยที่ต้องการเปลี่ยน
  - กด **Confirm Swap** เพื่อส่งคำสั่งไปยังเซิร์ฟเวอร์
  - เปลี่ยนได้เฉพาะกรณีที่เบี้ยในถุงเหลือ ≥ 8 ตัว (มีกติกาตรวจใน backend)
- **เวลานับถอยหลังของแต่ละคน**:
  - เวลาของผู้เล่นแต่ละคนจะลดลงตามจริง
  - หากหมดเวลา จะเริ่มมี **โทษล่าช้า (overtime penalty)** หักคะแนนทุก ๆ ช่วงเวลาที่เกิน (จัดการใน `server.mjs`)
- **ยอมแพ้ (Surrender)**:
  - กดจากเมนู → เกมจบทันที และอีกฝ่ายชนะ
- **จบเกม (Game Over)**:
  - เมื่อเงื่อนไขเกมจบถูกต้อง (เช่น เบี้ยหมดทั้งถุงและบนมือ) ระบบจะสรุปคะแนนและประกาศผู้ชนะ/เสมอ

### การแปลภาษา (TH / EN)

- ในหน้าเกม จะมีเมนู **Settings** (ไอคอนเฟือง)
- สามารถเลือก:
  - **ไทย (TH)**
  - **English (EN)**
- ข้อความทั้งหมดใน UI ที่เกี่ยวกับเกม เช่น “ตาของคุณ”, “คะแนน”, “ยอมแพ้”, “Victory” ฯลฯ อ่านค่าจาก `src/lib/translations.ts`

---

### การรันฝั่งเซิร์ฟเวอร์ / Socket

- ฝั่ง server ทั้งหมดอยู่ในไฟล์ `server.mjs`
  - สร้าง Express app
  - ติดตั้ง Socket.IO
  - จัดการห้อง (`rooms`), การสร้าง/เข้าห้อง, เริ่มเกม, ส่งตา, เปลี่ยนเบี้ย, ยอมแพ้, disconnect ฯลฯ
- Client เชื่อมต่อผ่าน `src/socket.ts` และใช้ events เช่น:
  - `createRoom`, `createBotRoom`, `joinRoom`, `startGame`, `makeMove`, `swapTiles`, `resign`, `getPublicRooms`
- การรันทั้งหมดถูกจัดการผ่านคำสั่งใน `package.json` ที่กล่าวไว้ด้านบน (`npm run dev`, `npm run build`, `npm run start`)

### สรุป

- ฝั่ง UI: `src/app/page.tsx`, `src/components/Home.tsx`, `src/components/Game.tsx`
- กติกา/คะแนน: `src/lib/tiles.js`, `src/lib/math_validator.js`, `src/lib/bot_ai.js`
- เซิร์ฟเวอร์: `server.mjs`
